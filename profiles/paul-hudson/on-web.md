# Paul Hudson — On Web

## Hackingwithswift.com as a Content Business

Hackingwithswift.com is the primary vehicle for Paul Hudson's business, community, and brand. It is not a blog or a portfolio — it is a content platform built around a specific, high-value topic (Swift and Apple platform development) and maintained consistently for over a decade. By 2019 it was generating more than 16 million page views per year, with organic search as the dominant traffic source.

The site's architecture serves discoverability: tutorials are written to answer the specific questions that developers search for, organized so that a query like "how to use List in SwiftUI" returns a complete, working answer with code. This is content marketing without the pretense — the content is genuinely useful, and being genuinely useful is how it ranks.

Hudson has explicitly encouraged linking to the site and recommending it on social media as the primary ways readers can support the work. This is consistent with how he thinks about web visibility: organic links and word-of-mouth referrals are more valuable than paid traffic, and earned authority through quality content is more durable than paid search placement.

## SEO as a Discovery Mechanism, Not a Manipulation

The site's SEO success is a byproduct of content quality rather than technical optimization as a primary goal. When a tutorial answers a specific Swift question completely — with working code, clear explanation, and accurate information for the current Swift version — it earns links and rankings naturally. Hudson's practice of updating tutorials for each new Swift and SwiftUI release means the content does not decay in relevance, which sustains rankings over time.

The solocoder.com podcast noted the site ranks in the top five Google results for Swift-related searches. This is the outcome of a decade of consistent, high-quality, frequently updated content rather than a shortcut.

## Written Tutorials vs. YouTube Video

Hudson produces both written tutorials and YouTube videos, and he has been explicit about why both formats exist rather than choosing one.

Written tutorials allow code to be copied. A developer working through a tutorial needs to move code into Xcode — video is the wrong format for that step. Written tutorials also have indefinite useful life; a reader can pause, re-read, and navigate non-linearly. For search and discoverability, written content indexes while video does not (except through descriptions and metadata).

Video serves a different purpose: it demonstrates technique in motion. Seeing a UI animate, or watching Hudson navigate Xcode while explaining a concept, communicates things that static code cannot. His tutorials at Hacking with Swift+ are delivered as both video and article for exactly this reason — the learner chooses the format that works for their current context.

His YouTube channel (youtube.com/@twostraws) carries tutorial playlists, the Swift on Sundays livestream archive, and series aligned with specific books and courses. The YouTube presence functions as a top-of-funnel for learners who discover his work through video search rather than web search.

## Ignite: Building the Web with Swift

In March 2024 Hudson released Ignite, an open-source static site generator that lets Swift developers build websites entirely in Swift using SwiftUI-like syntax. The project compiles to standard HTML, JavaScript, and CSS and includes accessibility support built into the output by default.

Ignite represents Hudson extending his platform expertise into web development tools — creating a path for Swift developers to build their own web presence without learning a separate language or framework. It is both a practical tool and a philosophical statement: Swift developers should not have to context-switch to build a professional website.

## Newsletter: Owned Audience

Hudson distributes a monthly newsletter (via Gumroad) covering Swift news, community updates, personal reflections, and site announcements. Monthly newsletters from 2023–2025 are publicly archived on his Gumroad page. The newsletter is a relationship with a subscribed audience rather than an algorithm-dependent social media audience — an owned channel that persists regardless of platform changes.
