# Amber Case — On Web

## The Calm Web vs. the Attention-Maximizing Web

Case's calm technology principles map onto web design with uncomfortable precision — because the web, as it evolved through the 2010s and 2020s, represents a near-complete inversion of every calm technology value. The modern attention-economy web is designed to maximize time-on-site, click-through rates, and notification opt-in rates. It uses modals, overlays, notification permission requests, cookie consent walls, email capture popups, autoplaying video, and infinite scroll — each of which is a deliberate interruption of the user's primary task. Combined, they constitute a hostile environment for any user trying to accomplish a specific goal.

The calm alternative is a web that serves the user's intent without exploiting their presence. This means: no interruptions that do not serve the user's current task, no dark patterns that trick users into unwanted commitments, no notifications that the user did not explicitly request, and no design that maximizes engagement metrics at the cost of user experience.

## Web Notifications as a Calm Technology Failure Case

Browser push notifications are, from Case's perspective, a canonical example of calm technology principles violated by default. The typical implementation:
- Requests permission at the worst possible moment (first visit, before the user has any basis for trust)
- Defaults to the highest-interruption channel (operating system-level alerts)
- Optimizes for opt-in rate rather than notification quality
- Treats any engagement (even dismissal) as a success metric

A calm web notification design would invert all of these defaults: request permission at the moment the user has demonstrated they want ongoing information from the site, default to a lower-interruption channel, set default notification frequencies conservatively, and measure success by user satisfaction rather than reach.

## Geoloqi and the Location-Aware Web

Case's direct contribution to web technology came through Geoloqi's developer platform, which provided infrastructure for location-based web services — geofencing, location triggers, and context-aware notifications delivered through web and mobile interfaces. Geoloqi's design philosophy applied calm technology principles to location: deliver the right information at the right location-triggered moment, without requiring the user to actively check in or monitor an app.

The calm promise of location-based web services is substantial: rather than asking users to remember to look things up (what's near me? what's open now?), location-aware services can surface relevant information at the precise moment it is needed. But this promise is easily corrupted by over-notification — location-aware products that send alerts for every proximity event replicate the same attention-tax problem as any other over-notifying product, with the additional dimension of physical movement as a trigger.

Case's Geoloqi approach — privacy-first, transient data, contextually triggered rather than continuous — represents the calm design version of location-based services.

## Web Design Implications of the Eight Principles

Applying Case's eight principles to web design produces specific guidance:

- **Minimal attention**: Page design that supports task completion without requiring users to manage the interface. Navigation that gets out of the way.
- **Inform and create calm**: Content pages that deliver information without making the reader manage advertising, related content, newsletter popups, and cookie notices before reaching what they came for.
- **Use the periphery**: Progress indicators, subtle status cues, and visual hierarchy that orient the user without demanding focus.
- **Minimum necessary technology**: Lean page weight, no unnecessary scripts, fast loads. Slow pages are attention-taxing pages.
- **Respect social norms**: No dark patterns. No pre-checked marketing opt-ins. No permission requests at first visit.

## Gap Note

Case has not published extensively on web design as a distinct domain. Her framework applies cleanly to web contexts, but specific web UX guidance must largely be inferred from her broader calm technology principles rather than drawn from dedicated web-focused writing or talks.
