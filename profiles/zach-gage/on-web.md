# Zach Gage — On the Web

## Puzzmo as Web-First Platform

Zach Gage spent the majority of his career designing for native iOS. Puzzmo (2023) represents his deliberate turn toward the web as a primary delivery mechanism for games — and his most developed thinking about what the web uniquely offers as a platform.

Puzzmo is built as a responsive web application, not a native app. Its layout is described as "a magazine page, with a selection of puzzle widgets laid out in columns that responsively scale to tablets and mobile devices." The choice to build web-first rather than app-first is significant: it removes the installation barrier, makes the platform immediately shareable by URL, and allows the experience to work across devices without requiring separate codebases.

## The Newspaper Model as Web Model

Gage's product vision for Puzzmo draws from the newspaper — specifically, the newspaper puzzle page as a content model that the web can replicate and improve upon. He observed that the newspaper created cross-discovery: a reader who opened the paper for the crossword might notice a word jumble next to it and try that too. The editorial curation of a puzzle page, displayed together, was a discovery engine.

The web replicates this through layout. Puzzmo's magazine-style front page shows multiple daily puzzles simultaneously, with progress visible from the main page — replicating "the experience of solving a newspaper's puzzle section and seeing the results memorialized in pencil." The web allows this to be social and persistent in ways print cannot be.

## Web vs. Native Apps for Games

Gage has not made extensive public statements contrasting web and native app development as a philosophical matter. His practical position can be inferred from Puzzmo's design decisions: the platform prioritizes responsiveness across screen sizes, shareable URLs, and no-install access over native performance or platform-specific features like full haptics.

The trade-off is real: native iOS games can access the full haptic engine, background audio, and OS-level integrations that web games cannot. Gage's earlier iOS games — Good Sudoku, Knotwords, SpellTower — were built to take advantage of these features explicitly. Puzzmo's puzzle games are simpler mechanically and can afford to run well in a browser context.

The subscription model also favors web: billing through a web platform avoids Apple's 30% App Store cut, a meaningful revenue consideration for a subscription business.

## Distribution and Reach

The Hearst Newspapers acquisition of Puzzmo positioned the platform as a newspaper digital product, distributed to Hearst's existing subscriber base and embedded in their digital properties. This is a native web distribution model — games delivered through editorial websites rather than through an app store. It mirrors how newspaper games have always worked, updated for the web.

Subsequent licensing to Paxton Media and Postmedia extended this distribution model further, with Puzzmo's games embedded in those publishers' digital editions. The web makes this kind of white-label, multi-publisher distribution straightforward in a way native apps cannot match.

## Gap Note

Gage has not made extensive public statements about web game performance, WebAssembly, WebGL, or browser-specific game development philosophy. His web views are primarily expressed through Puzzmo's design and business decisions rather than explicit technical or philosophical positions. Readers seeking strong opinions on game engine choices for the web should look elsewhere.
