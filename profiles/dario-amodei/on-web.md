# Dario Amodei — On Web

## Honest Gap Note

Amodei has not publicly discussed web presence, web strategy, or web product philosophy in any documented way. This file covers what can be inferred from Anthropic's actual web products and public communications posture.

## Anthropic's Web Presence

Anthropic's primary web properties are:

- **claude.ai** — the consumer and professional product interface, offering direct access to Claude across free, Pro, Max, Team, and Enterprise tiers
- **anthropic.com** — the company and research site, including the blog, research publications, model cards, safety documentation, and policy positions
- **console.anthropic.com** — the API developer console and playground
- **platform.claude.com** — API documentation and developer resources

Amodei's own writing appears at **darioamodei.com**, where he publishes long-form essays ("Machines of Loving Grace," "The Adolescence of Technology," "The Urgency of Interpretability") and shorter posts on AI policy.

## API Documentation as a Product

Anthropic's API documentation is notably detailed and developer-oriented, reflecting the enterprise and developer-first positioning that Amodei established from the outset. The documentation covers prompt engineering, system prompt design, Constitutional AI principles, and safety guidelines with more transparency than is common in the industry. This transparency is consistent with Amodei's philosophy: telling developers accurately what Claude can and cannot do builds the trust necessary for serious professional use.

## Claude Design

In April 2026, Anthropic launched Claude Design through Anthropic Labs — a web-native tool for generating visual assets, prototypes, presentations, and UI mockups. This extends Claude's capabilities into design workflow contexts and represents Anthropic's most direct move into design-tool territory (challenging products like Figma and Canva). It is powered by Claude Opus 4.7 and is currently in research preview.

## Communication Posture

Anthropic's blog and public communications consistently prioritize substance over marketing. Safety reports, model cards, research papers, and policy positions are published in detail unusual for a company at this stage of growth. This is consistent with Amodei's "show, don't tell" philosophy on safety: publish the work, explain the reasoning, let the record speak.
