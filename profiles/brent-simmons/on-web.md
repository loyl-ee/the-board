# Brent Simmons — On the Web

## RSS as a Web Philosophy

Simmons built his career on RSS, and RSS is not simply a file format — it is a position about how the web should work. An RSS feed is content that lives outside any single platform, that can be consumed by any compliant reader, that gives readers direct access to the sources they care about without the mediation of algorithms, advertising networks, or engagement-maximizing feeds.

He has described RSS as an expression of the open web: decentralized, standardized, user-controlled. When he decided in 2002 that NetNewsWire would not scrape web pages but would instead read RSS feeds, he was making a values choice as much as a technical one — he wanted to encourage publishers to adopt an open standard rather than building around proprietary content extraction.

## The Decline and Advocacy

Simmons has watched the open web lose ground to centralized platforms for two decades, and he has written about this loss with something close to grief. He told the micro.blog interview: "It has kind of made me sad these past five or ten years where it seems like development for the open web has slowed down a lot, particularly when we talk about the intersection of the Mac desktop and the open web."

He attributes part of this decline to developers' aversion to XML — a purely practical barrier that, in his view, made RSS less accessible to the generation of developers who came of age after JSON became the default serialization format. This diagnosis led directly to JSON Feed.

## JSON Feed: Lowering the Barrier

In May 2017, Simmons and Manton Reece announced JSON Feed, a syndication format built with JSON rather than XML. The stated goal was pragmatic: "I believe that developers (particularly Mac and iOS developers, the group I know best) are so loath to work with XML that they won't even consider building software that needs an XML parser. Which says to me that JSON Feed is needed for the survival of syndication." (inessential.com, "JSON Feed," May 17, 2017)

JSON Feed was also framed as a statement of intent about the open web: "What I'm hoping is that this is part of a bigger thing. The idea is to revitalize interest in the open web, in blogging, in syndication, and all of that kind of stuff." It is not accidental that Simmons chose to make inessential.com itself available as a JSON Feed.

## Against Centralization

Simmons' post "After Twitter" (inessential.com, November 13, 2022), written in the early days of Twitter's post-acquisition chaos, is his clearest statement on centralized platforms. "The internet's town square should never have been one specific website" with its own rules and its own owner. Twitter's dominance "could only deform the worldwide conversation, to disastrous ends." The platform "subdued creative energy" by making itself mandatory — anything built on the web was evaluated by whether it could replace Twitter.

He was energized by Twitter's decline, not despairing: "The web is wide open again, for the first time in what feels like forever." His vision is a return to the 2000s web — what he calls the "liveliness and sparkle" of that era — through modern technology. RSS, JSON Feed, micro.blog, Mastodon, and similar decentralized systems are his preferred tools for rebuilding what was lost.

## inessential.com as a 25-Year Web Presence

The blog is itself a web artifact worth noting. Simmons began inessential.com in November 1999 as a Manila blog while working at UserLand. It has been powered by multiple blog engines over the years and is now a static site generated by Ruby scripts — what he has called a "baked" site, after his 2011 essay advocating for static site generation over dynamic databases as a more reliable, faster, and traffic-spike-resistant approach to web publishing. (inessential.com, "A Plea for Baked Weblogs," March 16, 2011)

The blog embodies the decentralized web he advocates for: hosted on his own domain, powered by his own tools, with no third-party JavaScript or tracking, generating RSS and JSON feeds. It is what an independent web presence looks like when built by someone who believes in independent web presences.

## The Web and Mac Apps Together

Simmons has always been interested in the intersection of native Mac apps and the open web, not in choosing between them. NetNewsWire is a native Mac and iOS app that exists to serve a web standard. inessential.com is a website that documents the development of native apps. JSON Feed is a web format designed to be consumed by native readers. The open web and Mac-native software are not in tension for Simmons — they are complementary expressions of the same underlying value: user ownership and control over their own information.
