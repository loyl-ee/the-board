# Profiles Directory — Reader's Guide

This folder contains 33 expert profiles. Each profile is a folder with 11 topic files covering every aspect of how that person or studio thinks about building products, businesses, and experiences.

---

## How to Read a Profile

Each profile folder contains exactly these 11 files:

| File | What it covers | Best for |
|---|---|---|
| `overview.md` | Who they are, career arc, key works | First time reading someone new |
| `philosophy.md` | Core worldview and fundamental beliefs | Understanding their lens before applying it |
| `on-business.md` | Starting, growing, pricing, monetising | Business model and strategy questions |
| `on-product.md` | How they think about making software | Product direction, prioritisation, shipping |
| `on-design.md` | Visual design, aesthetics, craft | Design decisions, visual critique |
| `on-ux.md` | Interaction design, flows, onboarding | UX review, friction reduction |
| `on-customer-journey.md` | Acquisition, retention, trust | Marketing, support, loyalty |
| `on-web.md` | Web strategy, landing pages, distribution | Web presence, content, SEO |
| `on-safety.md` | Ethics, privacy, responsibility | Risk, compliance, ethical decisions |
| `quotes.md` | Verbatim sourced quotes | Their actual voice; use in presentations |
| `when-to-use.md` | When to invoke them, blind spots, pairings | Choosing who to consult |

**Always read `when-to-use.md` first** when you're unsure if someone is the right advisor for a question. It lists their blind spots and who they pair well with.

---

## How to Invoke Multiple Advisors

Read the same topic file across several people to get competing perspectives:

> "Read `on-business.md` from `dhh`, `pieter-levels`, and `marco-arment` — then give me three different takes on whether I should add a subscription tier."

> "Read `on-ux.md` from `jony-ive`, `amber-case`, and `amelia-wattenberger` — then critique this onboarding flow."

> "Read `quotes.md` from `steve-jobs` and `bill-campbell` — find the quotes most relevant to my team culture problem."

## Pairing Guide — Productive Tensions

| Should I blitzscale? | `reid-hoffman` | `dhh`, `marco-arment` |
