# Chris Eidhof — On Web

## objc.io as a Web Product

objc.io.io has been a web-first publication since its founding. All 24 monthly issues from 2013–2015 were published free on the web; the optional Newsstand app was a secondary access point, not the primary one. This was a deliberate editorial and business choice: make the content accessible to anyone with a browser, build the audience through openness, and monetize through optional premium tiers.

The website has been redesigned over the years but has maintained the same core function: a readable archive of articles, books, and video content organized around depth rather than recency. There is no algorithmic feed, no engagement-optimized layout. The design serves the content.

Swift Talk at talk.objc.io follows the same philosophy. The site is clean, organized by collections and episodes, and oriented around the subscriber relationship rather than discovery metrics. Half the episodes are publicly accessible; the subscription boundary is clear. This is a deliberate product decision: the web experience reflects the editorial values of the content itself.

## The SwiftUI Field Guide as an Interactive Web Product

Eidhof's most significant web product is the SwiftUI Field Guide (swiftuifieldguide.com, 2024), which reimplements key parts of SwiftUI's layout algorithm in TypeScript to create interactive, animated explanations of how the system works. This is a genuine web application — sliders, animations, dynamic type scaling — not just a static document.

The choice to build this as a web product rather than a native app is pragmatic: the audience is developers on any machine, and the interactivity requires a dynamic medium. Eidhof's stated motivation was frustration with static documentation: he wished SwiftUI's official docs would be "way more visual and interactive." The Field Guide is his answer to that wish, deployed on the web where it is accessible without installation.

## Personal Blog

Eidhof's personal blog at chris.eidhof.nl is his primary public writing outside objc.io. Posts range from technical deep dives (SwiftUI internals, struct semantics, type system design) to shorter reflections on projects, year-in-review posts, and occasional links. The blog runs on custom software — the GitHub repository for the blog itself is public (github.com/chriseidhof/chris.eidhof.nl), reflecting his preference for building and understanding his own tools.

Recent posts include technical explorations (a FormatStyle interactive guide in April 2026) and project retrospectives (Building Pancake in February 2026, on a Mac static-site app he built with a collaborator). The blog reads as a practitioner's notebook — direct, specific, without the editorial polish of objc.io articles.

## Views on Web vs. Native

Eidhof has not written extensively about the web vs. native platform debate as a general question. His career has been Apple-platform focused, and his content is for Apple platform developers. He has noted, via a casual Mastodon post, that he found himself wishing Apple's developer app made WWDC videos more accessible — a small signal about his preference for native delivery where it serves the audience.

His background includes building websites in PHP, HTML, and CSS before iOS, and his SwiftUI Field Guide demonstrates genuine web competency. But web development is a means to an end for him — a distribution mechanism for educational content — rather than a platform he advocates for or theorizes about. His views on this specific question are not a primary area of documented thinking.
